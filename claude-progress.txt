- **Issue**: When deleting records with foreign key constraints, SQLite was returning 500 errors instead of 400

- **Fix**: Updated error handling in 5 route files to also check for:
  - `error.code === 'SQLITE_CONSTRAINT'`
  - `error.code === 'SQLITE_CONSTRAINT_FOREIGNKEY'`
  - `error.message.includes('FOREIGN KEY')`
  - `error.message.includes('constraint')`

- State updates are asynchronous - multiple rapid clicks can all see saving=false
- Ref updates are synchronous - the first click sets it immediately
- The ref blocks all subsequent clicks in the same synchronous execution
