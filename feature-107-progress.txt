### Completed
- Feature #107: app_settings capability - IMPLEMENTED AND VERIFIED

### Implementation Summary
This feature adds a dynamic settings system to the SalesQuoteMgr application:

1. **Database Setup**
   - app_settings table already existed, seeded with initial data:
     - app_version: "1.0"
     - db_version: "1.0"
     - webapp_main_color: "blue"
     - index_page: "<h1>Welcome to SalesQuoteMgr</h1>"
     - client_name: "WAB Group USA"
   - Created sql/seed_app_settings.sql for documentation

2. **Backend API Endpoints**
   - GET /api/public-settings (unauthenticated) - returns client_name, app_version, db_version
   - GET /api/app-settings (authenticated) - returns webapp_main_color + versions + client_name
   - GET /api/app-settings/all (admin only) - returns all 5 settings
   - GET /api/app-settings/colors - returns list of 22 available Tailwind colors
   - PUT /api/app-settings/:name (admin only) - update a setting with validation

3. **Frontend Theming System**
   - Created utils/colorPalette.js with full Tailwind color palette (22 colors, 10 shades each)
   - Added CSS variables in index.css for dynamic theming
   - Created SettingsContext.jsx for global settings state management
   - applyThemeColor() function updates CSS variables at runtime

4. **UI Updates**
   - Login page: Shows client name "WAB Group USA" and version info "App: 1.0 | DB: 1.0"
   - Layout header: Shows client name below "SalesQuoteMgr"
   - Sidebar footer: Shows "App: 1.0 | DB: 1.0"
   - Admin-only "App Settings" menu item in sidebar

5. **Admin Settings Page** (/admin/settings)
   - View and edit all 5 settings
   - Theme Color: Dropdown + 22 clickable color swatches
   - Landing Page HTML: Textarea with live HTML preview
   - Save button only enabled when value is modified
   - Success/error feedback messages
